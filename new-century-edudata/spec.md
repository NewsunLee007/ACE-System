# ScoreAnalysis Data Capabilities Spec

## Why
目前教务系统的数据看板（Dashboard）和成绩分析（ScoreAnalysis）虽然具有基础的Z值模型，但缺乏专业数据专家的深度分析视角，板块间数据割裂且不够灵动。特别是在尖子生追踪、模拟进线分析（升学预测）以及历次考试的纵向进退步对比方面，系统未能提供直观的、具有Dashboard质感的可视化支撑。同时，对于教务处、班主任、家长三类核心角色的数据查看权限，缺乏系统级（菜单与路由层面）的严格隔离，存在越权查看数据的风险。

## What Changes
- **角色权限动态隔离**：重构前端路由和侧边栏渲染逻辑，基于用户角色（`role_name` / `permission_code`）动态下发菜单权限，实现“教务大屏、班主任视图、家长门户”的严格物理隔离，权限外的菜单不可见。
- **数据源灵动导入**：在成绩分析模块新增数据导入录入区，同时支持“Excel文件上传”和“剪贴板直接粘贴解析（Smart Paste）”，涵盖每次考试的明细数据（考号、班级、各科原始分）以及历史考试汇总数据（参考人数、上线率等）。
- **尖子生与分层分布追踪**：
  - 新增“尖子生追踪表格”，自动提取年级总分前50/100名学生名单，支持表头固定、多维度排序及按班级过滤。
  - 新增“高分段分布对比柱状图”，直观展示前50/100/200名在各班级的人数分布。
- **“三率一分”自动化报表**：分班级自动计算并渲染语文、数学、英语、科学、社会及总分的“优秀率(前20%)”、“及格率(>60%)”、“平均分”和“后20%率”，并高亮每项指标排名第一的班级。
- **模拟进线预测（核心）**：
  - 支持手动输入“重高模拟分数线”和“普高模拟分数线”。
  - 系统自动反算各班级的“进线人数”、“上线率”（保留两位小数加%）。
  - 新增“学科贡献度”雷达图或条形图，剖析单科对进线学生的拉动作用。
- **纵向历史对比看板（弹窗/抽屉）**：
  - 在录入历史数据（含历次上线率）与最近一次考试数据后，提供“计算差值”按钮。
  - 点击按钮触发侧边抽屉（Drawer）或全屏模态框（Modal）展示对比结果。
  - 自动计算最新上线率与历次上线率的差值，正数（进步）以绿色/红色向上箭头标识，负数（退步）以反色向下箭头标识，并在最下方合计全年级总参考人数与总上线率。

## Impact
- Affected specs: UI 布局架构、数据导入解析机制、成绩统计引擎、路由权限守卫。
- Affected code: 
  - `frontend/src/App.jsx` & `Layout.jsx` (路由与权限控制重构)
  - `frontend/src/components/ScoreAnalysis.jsx` (主控重构)
  - 新增 `frontend/src/components/ScoreAnalysis/` 目录存放细分的数据可视化子组件（如进线预测、三率一分、历史对比抽屉等）。

## ADDED Requirements
### Requirement: 动态权限与隔离
The system SHALL provide 严格的角色路由守卫。
#### Scenario: 班主任越权访问
- **WHEN** 班主任（非教务处管理员）尝试通过 URL 直接访问 `/dashboard` 或在侧边栏寻找该入口。
- **THEN** 侧边栏不渲染该菜单，且路由守卫将其重定向至 `/headteacher` 视图，并提示无权限。

### Requirement: 模拟进线与学科贡献度分析
The system SHALL provide 动态的模拟分数线划定与多维归因分析。
#### Scenario: 手动划线测算
- **WHEN** 教务主任在分析面板输入“重高线: 580, 普高线: 520”。
- **THEN** 系统立即刷新各班级的进线人数和上线率，并生成五大学科对过线学生的贡献度图表。

### Requirement: 历史差值自动对比抽屉
The system SHALL provide 直观的历次考试纵向进退步对比。
#### Scenario: 触发差值计算
- **WHEN** 用户在录入区准备好历史数据与本次数据后，点击“计算差值”按钮。
- **THEN** 从屏幕右侧滑出抽屉（Drawer），并排对比各班级历次与本次的上线率，差值自动标红（或标绿）并附带升降箭头，底部生成年级大盘合计数据。

## MODIFIED Requirements
### Requirement: 成绩分析看板 UI
原有散落在 `ScoreAnalysis` 中的简单表格将被替换为基于 `shadcn/ui` 的专业级 Dashboard 质感组件（Card, DataTable, Drawer），排版更紧凑，全面支持表头固定和字段排序，满足会议投屏的高要求。
