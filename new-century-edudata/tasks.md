# Tasks

## Task 1: 权限控制与侧边栏动态路由
- [ ] SubTask 1.1: 在 `App.jsx` 和 `Layout.jsx` 中，基于登录用户的 `role_name` 实现菜单和路由的严格权限控制（管理员、教务处主任可见全局大屏；班主任仅见班级视图和成绩分析；家长仅见家长门户）。
- [ ] SubTask 1.2: 补充全局路由拦截器（ProtectedRoute），对无权限访问特定页面的请求进行重定向并显示友好提示。

## Task 2: 成绩分析看板基础架构升级 (ScoreAnalysis)
- [ ] SubTask 2.1: 重构 `ScoreAnalysis.jsx` 为现代化 Dashboard 布局（采用 `shadcn/ui` 的 Tabs, Card 等组件），规划四大 Tab：数据录入、基础统计(三率一分)、进线预测、纵向对比。
- [ ] SubTask 2.2: 实现一个通用的“Smart Paste”组件，支持从 Excel 直接粘贴或上传文件导入考试成绩清单和历史对比数据（包含班级、参考人数、上线率等）。

## Task 3: 尖子生追踪与分层分布
- [ ] SubTask 3.1: 开发“尖子生追踪模块”表格，自动提取全科总分前50名、前100名，支持表头固定和按班级快速过滤。
- [ ] SubTask 3.2: 开发“高分段分布对比”柱状图，统计并展示前50/100/200名在各班级的分布情况。

## Task 4: “三率一分”统计报表
- [ ] SubTask 4.1: 开发按班级、按学科统计的“三率一分”表格（优秀率前20%、及格率>60%、平均分、后20%率）。
- [ ] SubTask 4.2: 为表格添加条件渲染逻辑，高亮显示各学科指标排名第一的班级数据。

## Task 5: 模拟进线分析 (核心)
- [ ] SubTask 5.1: 在看板中提供输入框，支持同时手动输入“重高模拟分数线”和“普高模拟分数线”。
- [ ] SubTask 5.2: 编写计算引擎，根据输入的模拟线自动计算各班的“进线人数”和“上线率”（保留两位小数加%），并在表格下方汇总全年级总参考人数、总进线人数和总上线率。
- [ ] SubTask 5.3: 开发“学科贡献度”分析图表（雷达图或条形图），剖析上线学生中各单科成绩对总分的拉动作用。

## Task 6: 历史差值自动对比抽屉 (核心)
- [ ] SubTask 6.1: 集成 `shadcn/ui` 的 Drawer (抽屉) 或 Dialog 组件，作为纵向对比看板的容器。
- [ ] SubTask 6.2: 在主面板提供“计算差值”触发按钮，点击后提取历次上线率和本次上线率并排显示在抽屉表格中。
- [ ] SubTask 6.3: 编写差值计算逻辑，将（最新上线率 - 历次上线率）的结果用正负号、带颜色（红绿）的向上/向下箭头标识，实现一目了然的进退步对比。
